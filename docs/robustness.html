Variance-method benchmark (Web Appendix G)

On the smallest NHANES domain (E3), six routes to the standard error of the same fitted cross-fitted estimator agree to within 3%, while the only route that re-fits the machine-learning nuisances (the full cluster bootstrap) costs about 15 million times more core-seconds — so the design-based linearization standard error is both valid and essentially free. The compute landscape shows that the primary cross-fitted estimator is computationally practical, not V times more costly than a single fit. (Web Tables S35–S36.)

## Finite-population correction

Omitting the finite-population correction (FPC) does **not** over-cover when the
sampling fraction is non-negligible — coverage is calibrated and flat in the
fraction, confirming the no-correction variance is the safe, conservative choice.
*(Web Table S25)*

::: {.panel-tabset}
#### Figure
```{r fpc_fig}
#| fig-height: 4
#| fig-alt: "Coverage by rung for the Fully-Aware and Fully-Aware-CF arms with and without the finite-population correction; coverage is near nominal either way."
coverage_fig("R10_fpc_partA_summary.csv", facet = "fpc",
             facet_labels = c(no = "no FPC", yes = "with FPC"))
```

## Variance-method benchmark (Web Appendix G)

On the smallest NHANES domain (E3), six routes to the standard error of the
**same** fitted cross-fitted estimator agree to within 3%, while the only route
that re-fits the machine-learning nuisances (the full cluster bootstrap) costs
about 15 million times more core-seconds — so the design-based linearization
standard error is both valid and essentially free. The compute landscape shows
that the primary cross-fitted estimator is computationally practical, not *V*
times more costly than a single fit. *(Web Tables S35–S36.)*

### Six variance routes (Web Table S35)

::: {.panel-tabset}
#### Figure
```{r bench_variance_fig}
#| fig-height: 3.2
suppressMessages({ library(ggplot2); library(plotly) })
bv <- read_csv_app("bench_variance.csv"); bv$method <- factor(bv$method, levels = rev(bv$method))
p <- ggplot(bv, aes(se, method, text = sprintf("%s\nSE = %.4f (SE/lin %.2f)\nwall %s s", method, se, se_ratio, wall_s))) +
  geom_vline(xintercept = 0.0199, linetype = "dashed", colour = "grey55") +
  geom_segment(aes(x = 0.0199, xend = se, yend = method), colour = "#0072B2", linewidth = 0.5) +
  geom_point(size = 2.6, colour = "#0072B2") +
  labs(x = "Standard error of the same Fully-Aware-CF fit (E3)", y = NULL,
       subtitle = "Dashed line: linearization SE. All six routes agree within 3%.") +
  theme_minimal(base_size = 12)
ggplotly(p, tooltip = "text")
```

### Compute landscape (Web Table S36)

::: {.panel-tabset}
#### Figure
```{r bench_compute_fig}
#| fig-height: 3.6
suppressMessages({ library(ggplot2); library(plotly) })
bc <- read_csv_app("bench_compute.csv"); bc$estimator <- factor(bc$estimator, levels = rev(bc$estimator))
p <- ggplot(bc, aes(total_s, estimator, text = sprintf("%s\ntotal %.1f s (V=%.0f, refits=%.0f)", estimator, total_s, as.numeric(V), as.numeric(refits)))) +
  geom_segment(aes(x = 0.1, xend = total_s, yend = estimator), colour = "#117733", linewidth = 0.5) +
  geom_point(size = 2.6, colour = "#117733") + scale_x_log10() +
  labs(x = "Total compute time (s, log scale)", y = NULL,
       subtitle = "The primary cross-fitted estimator (~34 s) is cheaper than the single-fit arms (112–194 s).") +
  theme_minimal(base_size = 12)
ggplotly(p, tooltip = "text")
```
